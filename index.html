Creating menus in HTML5

In HTML5 a menu is a list of commands, not just ordinary list elements, but this is still not supported by all browsers (1).

<menu> and <ul> are both used to set up unordered lists of items

  • First
  • Second

          <ul>
        <li> <a href="#"> First </a></li>
        <li> <a href="#"> Second </a></li>
        <li> <a href="#"> Third </a></li>
      </ul>

But you can also use the <menu> tag when you have a list of commands 

        
      <menu type="popup">
      <menuitem label="New" onclick="newFile()">
      <menuitem label="Save" onclick="saveFile()">
      <menuitem label="Edit" onclick="editFile()">
     </menu>

You can also use <menuitem> to label items in your menu 

      
    <menu>
   <menu label="Share on...">
    <menuitem label="Twitter" icon="ico_twitter.png"
    onclick="window.open('//twitter.com/intent/tweet?text='+window.location.href);">
    </menuitem>
    <menuitem label="Facebook" icon="ico_facebook.png"
    onclick="window.open('//facebook.com/sharer/sharer.php?u='+window.location.href);">
    </menuitem>
    </menu>

Type attributes for menu tag

There are three type attributes for the menu tag (2):

  • List: A list of commands or elements
  • Context: Must be activated - a pop-up menu
  • Toolbar: Active commands

List

This is a list of commands that the user can choose to perform and uses the -li tag

Context

This type of menu can only be accessed by an action taken by the user - clicking on a button, right clicking, etc.

Toolbar

Place to present active commands that the user can perform

As the menu tag is still not functional in many browsers, you can expect the following default settings (3)

    When to use <menu>

    <menu> should NOT be used as a navigation tool

    <menu> is for interaction with the website
